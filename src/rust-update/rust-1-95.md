# Rust 1.95.0 มาแล้ว

> 📅 วันที่เผยแพร่: 2026-04-20

มาแล้วกับ Rust 1.95.0 (อัปเดตกันได้ผ่าน $ rustup update stable) แม้ว่าเวอร์ชันนี้อาจจะไม่ได้มีฟีเจอร์ใหญ่อะไรมากนัก แต่ถ้าเรามองดีๆจะเห็นถึงความพยายามของทีมพัฒนาในการเกลาตัวภาษาให้เขียนสนุกขึ้น ลดความซ้ำซ้อน และเพิ่มความสะดวกในการจัดการเมมโมรี่ในระดับที่ลึกขึ้น

ผมขอเริ่มจากจุดที่เชื่อว่าคนทำระบบที่ต้องรันหลายแพลตฟอร์มน่าจะถูกใจ ปกติเวลาเราเขียนโค้ดแยกตามระบบปฏิบัติการหรือจัดการฟีเจอร์แฟล็ก (Feature flags) ที่ซับซ้อน เรามักจะต้องพึ่งพา crate ยอดฮิตอย่าง `cfg-if` มาช่วยจัดการ แต่ใน Rust 1.95.0 นี้ ทีมงานได้ยกแนวคิดนี้เข้ามาไว้ใน standard library โดยตรงผ่านมาโคร `cfg_select!` ความน่าสนใจคือมันถูกออกแบบมาให้ใช้ซินแทกซ์ที่คล้ายกับการเขียน `match` ซึ่งช่วยให้อ่านง่ายกว่าโค้ดสมัยก่อนมาก โดยตัวมาโครนี้จะประมวลผลเงื่อนไขตั้งแต่ตอนคอมไพล์ (Compile-time) และทำการ expand เฉพาะบล็อกแรกที่เงื่อนไขประเมินค่าออกมาเป็นจริงเท่านั้น ถือเป็นการบอกลา dependency ไปได้อีกหนึ่งตัวแบบหล่อๆ 😎 แถมทำให้โค้ดที่ต้องผูกติดกับ OS ดูสะอาดตาขึ้นมาก

เมื่อพูดถึงซินแทกซ์สไตล์ `match` แล้ว อีกหนึ่งการอัปเกรดที่ต่อเนื่องกันคือการทำงานของ Pattern Matching ที่ยืดหยุ่นขึ้น ต่อยอดจากตอนที่ Rust 1.88 เปิดตัวการใช้ let chains ตอนนี้ความสามารถนั้นถูกนำมาใช้ใน `match` expressions เรียบร้อยแล้ว นั่นแปลว่าเราสามารถใส่ `if let` เข้าไปเป็น guard ต่อท้ายเงื่อนไขในแต่ละ arm ได้เลย สิ่งนี้ช่วยแก้ปัญหาโค้ดทรงสามเหลี่ยมที่เราต้องเขียน `if let` ซ้อนเข้าไปข้างในอีกชั้นเวลาที่ต้องการสกัดค่าออกมาคำนวณต่อ ทำให้โค้ดเป็นระเบียบขึ้น แต่อย่าลืมจุดสังเกตสำคัญครับว่า คอมไพเลอร์จะยังไม่ได้เอาเงื่อนไขใน `if let` เหล่านี้ไปคำนวณเพื่อตรวจสอบว่าเราเขียนครอบคลุมทุกเคสแล้วหรือยัง ดังนั้นเราก็ยังต้องเตรียม fallback arm หรือใส่ `_ => {}` ปิดท้ายไว้เพื่อกันเหนียวเช่นเดิม

ขยับลึกลงมาอีกนิด รอบนี้มีการประกาศให้ API ที่จัดการเมมโมรี่กลายเป็น stable หลายตัวมาก นี่สวรรค์ของคนที่ต้องไปคลุกคลีกับ Unsafe Rust ชัดๆ เริ่มตั้งแต่กลุ่มของ `MaybeUninit` ที่เปิดให้เราแปลงประเภทข้อมูลของ Array ที่ยังไม่ได้จองค่าในเมมโมรี่ (Uninitialized memory) ได้สมูทขึ้นผ่าน `AsRef`, `AsMut` หรือแปลงข้ามเป็น `Cell` ได้เลย ซึ่งช่วยให้การสร้าง custom allocator ปลอดภัยและลดการเขียนแคสต์แปลกๆ ที่เสี่ยงต่อการเกิดพฤติกรรมที่ไม่คาดคิด (Undefined Behavior) นอกจากนี้กลุ่มของการดัดแปลงตัวชี้ (Raw Pointers) ก็มีฟังก์ชันตัวช่วยอย่าง `as_ref_unchecked` และ `as_mut_unchecked` ออกมาให้เรียกใช้ ช่วยอธิบายเจตนาการทะลวงข้อจำกัดของโค้ดได้ชัดเจนขึ้นผ่านชื่อเมธอด

ในมุมของโครงสร้างข้อมูลพื้นฐาน (Collections) อย่าง `Vec`, `VecDeque` หรือ `LinkedList` ก็มีการเพิ่มเมธอดตระกูล `_mut` เข้ามา เช่น `push_mut` หรือ `insert_mut` ความเจ๋งคือพอเรายัดข้อมูลลงไปปุ๊บ มันจะคืนค่ากลับมาเป็น Mutable reference ของค่านั้นให้เราเอาไปปรับแต่งต่อได้ทันที ไม่ต้องมานั่งเขียนโค้ดดึงตัวสุดท้ายออกมาก่อนให้เหนื่อยเปล่า ส่วนใครที่ชอบจับตรรกะไปรันตอนคอมไพล์ (Const context) ฟังก์ชันอย่าง `fmt::from_fn` หรือกลุ่มเช็คสถานะใน `ControlFlow` อย่าง `is_break` และ `is_continue` ก็สามารถเรียกใช้งานแบบ `const` ได้แล้ว

แต่ก็ใช่ว่าจะมีแต่การเดินหน้านะครับ ฟีเจอร์บางอย่างที่ไปต่อไม่ไหวก็ถูกถอยกลับมาตั้งหลักใหม่ อย่างเช่นการส่งสเปคของ Custom Target รูปแบบไฟล์ JSON ให้กับ `rustc` ตอนนี้ทีมงานตัดสินใจถอดการรองรับบนเครื่องมือฝั่ง stable ออกไปแล้ว (Destabilized) ใครที่ทำสาย Embedded หรือพัฒนา OS ของตัวเองจะต้องกลับไปรันบน Nightly แทน ซึ่งลึกๆ แล้วเหตุผลของทีมงานก็สมเหตุสมผลครับ เพราะการจะบิลด์ standard library อย่าง `core` ด้วยสเปคแบบ Custom มันก็มักจะถูกบังคับให้ต้องใช้ฟีเจอร์ของ Nightly อยู่แล้ว การเปิดไว้บน Stable จึงไม่ค่อยตอบโจทย์นัก ซึ่งตอนนี้ทางทีมก็กำลังเปิดรับฟัง Use case ต่างๆ บน Tracking issue เพื่อหาจุดลงตัวในอนาคตครับ
